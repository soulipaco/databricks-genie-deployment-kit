# 03 Playbook Ingestion and Chunking

## Notebook

Playbook ingestion is handled by `setup/vector_search.py`.
It is idempotent — re-running it upserts chunks via MERGE and syncs the VS index.

## PDF Source Types

The notebooks support two source types, controlled by `pipeline_config.yml`:

### source_type: volume

PDFs are read from a UC Volume path:
```yaml
playbooks:
  source_type: "volume"
  volume_path: "/Volumes/my_catalog/my_schema/my_volume"
```

Use this in production. Upload PDFs first with `setup/upload_pdfs.py`.

### source_type: local

PDFs are read from the workspace repo path:
```yaml
playbooks:
  source_type: "local"
  local_path: "pipeline_playbook_generator/generated/pdfs"
```

Use this for quick iteration before setting up a Volume, or for testing.
The PDFs must be committed to the Databricks Repo or generated in-place.

## PDF Registry

The registry of which PDFs map to which `question_category` lives in:
```yaml
playbooks:
  sources:
    - pdf_file: "demo_room_sales_performance_action_playbook.pdf"
      question_category: "sales_performance"
```

Every category used in `questions.seeds` must have a matching entry here.

## Where PDFs Come From

PDFs are generated by the playbook generator:
```
python pipeline_playbook_generator/generate_playbook.py --kit-format genie_kit
```

The generator reads domain content from `pipeline_playbook_generator/config/playbook_blueprint.yml`
and kit metadata from `geniecode/`, `metadata/columns/`, and `room.config.yml`.

Output lands in `pipeline_playbook_generator/generated/pdfs/`.

## Chunking Strategy

The chunker splits PDF text into overlapping character windows:
- Chunk size: 800 characters
- Overlap: 100 characters
- Each chunk gets a deterministic `chunk_id` (MD5 of `filename::chunk_index`)

The overlap prevents semantic content from being split across chunk boundaries.

## Required Playbook Structure

For chunking quality, playbook PDFs should follow this structural pattern
(generated automatically by `generate_playbook.py`):

```
1) KPI Reference ...
2) Scenario Playbooks ...
   Scenario A: ...
   Scenario B: ...
3) Recommendation Templates ...
   Action Card: ...
4) Putting It Together ...
   Example Plan 1: ...
   Example Plan 2: ...
```

Changing heading families without updating the generator will reduce chunk coherence.

## Stable Chunk Schema

Each row in `{prefix}_playbook_chunks`:

| Column | Description |
|---|---|
| `chunk_id` | Deterministic MD5 — enables idempotent MERGE |
| `source_file` | PDF filename |
| `question_category` | Category for VS filter |
| `chunk_index` | Position within the source file |
| `chunk_text` | Raw text of the chunk |

The Vector Search index filters on `question_category` — every source entry must
set this correctly or retrieval will return empty results.

## Practical Rule for Adding a New Playbook

1. Add domain content to `pipeline_playbook_generator/config/playbook_blueprint.yml`.
2. Run `generate_playbook.py --kit-format genie_kit` to produce the PDF.
3. Upload the PDF: `python setup/upload_pdfs.py` (if using `source_type: volume`).
4. Add a registry entry under `playbooks.sources` in `pipeline_config.yml`.
5. Add a matching entry under `questions.seeds` and `system_prompts` in `pipeline_config.yml`.
6. Re-run `setup/vector_search.py` in Databricks to update chunks and sync the index.
